# Work 지침: 2048 게임 구현

## 참고 문서
구현 전에 반드시 `D:\project\my_blog\spec.md`를 전체 읽고, 거기 설계된 내용을 그대로 따라 구현해라. 설계를 바꾸고 싶은 부분이 있어도 임의로 바꾸지 말고 spec.md에 쓰인 대로 구현해라.

## 수정 가능 범위 (이 파일들만 만들거나 수정한다)
- `src/games/2048/index.html` (신규 생성)
- `src/games/2048/style.css` (신규 생성)
- `src/games/2048/game.js` (신규 생성)
- `build/config.js` (기존 파일 수정 — nav 배열에 2048 항목 추가만)
- `build/build.js` (기존 파일 수정 — `copyStaticAssets()`에 games 복사 블록 추가만)

위 5개 파일 외에는 어떤 파일도 만들거나 수정하지 마라. 특히 `posts/`, `build/frontmatter.js`, `build/markdown-parser.js`, `build/templates.js`, `src/styles/main.css`, `src/scripts/theme-toggle.js` 등은 절대 건드리지 마라.

## 구현 요구사항 (spec.md 요약, 세부는 spec.md 원문 기준)
1. **HTML**: spec.md 3절의 마크업 구조를 그대로 사용 (헤더/네비/점수판/보드/오버레이). 모든 자산·링크는 상대 경로.
2. **게임 로직(game.js)**: 4x4 보드, "왼쪽으로 밀기" 함수를 회전/반전으로 재사용해 4방향 처리, 병합 시 3연속 병합 방지, 이동 후에만 새 타일 생성, 승리(2048)/패배(더 이상 이동 불가) 판정, 키보드 방향키 입력(`event.preventDefault()` 필수), 터치 스와이프 입력(모바일 대응), 점수/최고점수(localStorage 키 `game2048-best-score`) 관리, 새 게임 버튼.
   - 코드 스타일: `src/scripts/*.js`와 동일하게 IIFE + `var` + `function () {}`, `'use strict'` 없음. 주석은 자명하지 않은 이유가 있을 때만 최소한으로.
3. **CSS(style.css)**: `main.css`의 다크모드 3단 구조(`:root` 기본값 → `@media (prefers-color-scheme: dark)` → `:root[data-theme='dark']`/`:root[data-theme='light']`)를 타일 색상 변수에 동일 적용. 반응형 정사각형 보드(`min(90vw, 480px)`), `touch-action: none`, 타일 이동 트랜지션.
4. **build/config.js**: `nav` 배열에 `{ label: '2048', href: '/games/2048/' }` 추가.
5. **build/build.js**: `copyStaticAssets()`에 `src/games` → `dist/games` 복사 블록 추가 (기존 styles/scripts 복사와 동일 패턴, `fs.existsSync` 체크 포함).
