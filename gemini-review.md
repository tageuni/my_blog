# 요약

기존 리뷰 문서(`review.md`, `review-pixel-editor-report.md`)에서 발견한 2048 게임의 애니메이션 및 로직 버그, 픽셀 아트 에디터의 이벤트 처리 누락 등을 제외하고, 프로젝트 전체의 아키텍처, 보안, 성능, 접근성(A11y), 코드 품질 관점에서 코드베이스를 분석했습니다.

정적 사이트 제너레이터(Node.js 빌드 스크립트)와 바닐라 JS 기반 독립 웹앱들이 요구사항에 맞춰 간결하게 잘 구현되어 있으나, **보안(XSS) 취약점**과 블로그 스케일 아웃 시 발생할 **성능 병목**, 그리고 시각 장애인 및 키보드 사용자를 위한 **접근성(A11y) 결함**을 새롭게 발견했습니다.

## 발견된 문제

### 1. [보안] Markdown 파서의 링크/이미지 XSS (Cross-Site Scripting) 취약점
- **파일:라인**: `build/markdown-parser.js` (148~158 라인, `parseInline` 함수)
- **재현/근거**: `escapeHtml`을 통해 기본 HTML 태그 삽입은 방어하고 있으나, 마크다운 링크 `[텍스트](url)`와 이미지 `![대체텍스트](url)` 구문 파싱 시 `url` 부분의 스킴(scheme)을 전혀 검증하지 않습니다. 만약 마크다운에 `[클릭](javascript:alert(1))`이라고 작성하면 그대로 `<a href="javascript:alert(1)">클릭</a>`으로 렌더링됩니다. 정적 블로그 특성상 당장은 관리자 본인만 코드를 작성해 위험도가 낮아 보이지만, 외부 기고를 받거나 CMS를 연동할 경우 심각한 보안 홀이 됩니다.
- **심각도**: **High**

### 2. [성능/확장성] 검색 인덱스 파일 크기 무제한 증가 (네트워크/OOM 문제)
- **파일:라인**: `build/build.js` (121~130 라인), `src/scripts/search.js`
- **재현/근거**: 빌드 스크립트가 모든 포스트의 본문(HTML을 제거한 전체 평문 텍스트)을 `search-index.json` 배열에 그대로 담아 생성합니다. 이후 사용자가 검색 입력창을 포커스하는 순간 `search.js`가 이 파일을 통째로 다운로드합니다. 향후 블로그 글이 수백 개 이상으로 쌓일 경우 JSON 파일 용량이 수 MB에 달하게 되어, 트래픽 낭비는 물론 모바일 브라우저에서 JSON 파싱 시 메인 스레드를 블로킹(UI 멈춤)하는 성능 병목을 유발합니다.
- **심각도**: **Medium**

### 3. [접근성] 키보드 포커스 링(Focus Ring) 스타일 누락
- **파일:라인**: `src/styles/main.css`
- **재현/근거**: 프로젝트 전체 CSS에 `:focus`나 `:focus-visible` 상태에 대한 명시적 스타일이 전무합니다. 링크, 버튼(예: 테마 토글, 2048 새 게임, 픽셀 아트 툴바), 검색창 등을 Tab 키로 탐색할 때 브라우저의 기본 아웃라인에 의존하게 되는데, 이는 커스텀 배경색이나 다크모드에서 대비가 무너져 보이지 않을 수 있습니다. 키보드 전용 사용자에게는 현재 초점이 어디 있는지 파악하기 매우 어렵게 만듭니다.
- **심각도**: **Medium**

### 4. [접근성] 동적 검색 결과 스크린 리더 알림 누락 (`aria-live`)
- **파일:라인**: `build/templates.js` (66 라인), `src/scripts/search.js` (29~56 라인)
- **재현/근거**: 사용자가 검색창에 타이핑하면 JS가 `<ul id="search-results">`의 내부 DOM을 동적으로 채우고 `hidden` 속성을 제거하여 결과를 노출합니다. 그러나 해당 컨테이너에 `aria-live="polite"`나 `role="status"` 같은 속성이 선언되어 있지 않아, 시각 장애를 가진 스크린 리더 사용자는 검색 결과가 화면에 나타났는지, 검색 결과가 몇 개인지 음성 피드백을 받을 수 없습니다.
- **심각도**: **Medium**

### 5. [품질/SEO] 파비콘(Favicon) 누락 및 메타 데이터 부족
- **파일:라인**: `build/templates.js` (41~84 라인, `layout` 함수)
- **재현/근거**: 생성되는 모든 HTML의 `<head>` 태그에 `<link rel="icon">`가 정의되어 있지 않고 물리적인 `favicon.ico` 에셋도 존재하지 않아 웹 브라우저 콘솔 및 서버 로그에 지속적으로 404 에러가 발생할 수 있습니다. 더불어 `Open Graph(og:)` 메타 태그나 `Twitter Card` 태그가 누락되어, 작성한 글을 SNS나 메신저로 공유할 때 썸네일과 설명이 올바른 카드로 렌더링되지 않는 SEO 품질 저하가 있습니다.
- **심각도**: **Low**

## 개선 제안

1. **마크다운 XSS 방어**: `parseInline`에서 정규식으로 매칭된 `url`이 `http://`, `https://`, `mailto:`, `/`, 혹은 `./` 등으로 시작하는지 검사하거나 화이트리스트 기반의 URL 새니타이즈 로직을 추가하세요.
2. **검색 인덱스 경량화**: `search-index.json`에 본문 전체(`post.content`)를 넣지 말고 맨 앞 200~300자 정도의 발췌문(excerpt)만 포함시키거나, Lunr.js/Fuse.js 처럼 서버사이드나 빌드 타임에 미리 역색인(Inverted Index)을 구성하여 클라이언트로 내려주는 방식으로 구조를 개선하세요.
3. **포커스 가시성 확보**: `main.css`에 `a:focus-visible, button:focus-visible, input:focus-visible { outline: 2px solid var(--accent); outline-offset: 2px; }` 등 일관된 포커스 스타일을 추가하세요.
4. **접근성(A11y) 속성 추가**: `<ul id="search-results" aria-live="polite">` 속성을 추가하여 검색 상태를 보조기기가 읽을 수 있게 개선하세요.
5. **메타데이터 보강**: 루트 디렉토리에 기본 파비콘을 추가하고, `templates.js`의 `layout` 함수에 `og:title`, `og:description`, `og:url` 등 필수 소셜 메타 태그를 주입하도록 설정하세요.
